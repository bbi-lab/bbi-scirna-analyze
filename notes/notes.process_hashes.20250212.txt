== Run process hashes when channel is not empty.

Note: more generally, drop empty channel entries

Based on the Nextflow documentation, there may be several possible solutions.

o  the 'when' directive is 'deprecated' and the Nextflow documentation suggest
   using an if block in the workflow or the filter operator in the channel.
   There is also an ifEmpty() operator which may be used in tandem with the
   filter operator.

o  how to detect an empty groovy map or channel:
     o  the find() 'interface' (https://docs.groovy-lang.org/docs/next/html/groovy-jdk/java/util/Collection.html#find())

o  Nextflow filter:
   Returns: queue channel or value channel, matching the source type

   The filter operator emits the items from a source channel that satisfy
   a condition, discarding all other items. The filter condition can be a
   literal value, a regular expression, a type qualifier (i.e. Java class),
    or a boolean predicate.

o  Nextflow ifEmpty():
   Returns: queue channel or value channel, matching the source type

   The ifEmpty operator emits a source channel, or a default value if the
    source channel is empty (doesn't emit any value):


o  using filter to check for non-empty channel

     https://github.com/nextflow-io/nextflow/issues/526

     process demuxByPrimers {
       publishDir params.outdir

       input:
       file productFile from products.filter{ it.size()>0 }

       output:
       file 'ITS*.fastq' into primers mode flatten

       script:
       println "Processing ${productFile}"
     }
