== STARsolo soloCellReadStats file columns

url: https://github.com/alexdobin/STAR/issues/1501

question:

I understand that :
cbMatch        means the number of read that matched the cell barcode,
cbPerfect      for the number of perfect match on cell barcode
exonic         for the number of read mapping on exonic
intronic       for the number of read mapping on intronic
mito           for the number of read mapping on mitochondrial genome
nUMIunique     for the total number of counted UMI
nGenesUnique   for the number of genes having non 0 counts (not sure for this one)

answer:

your descriptions of the fields are all correct!
Other fields:
genomeU      : number of reads mapping to one locus in the genome
genomeM      : number of reads mapping to multiple loci in the genome
featureU     : number of reads mapping to one feature (Gene, GeneFull, etc)
featureM     : number of reads mapping to multiple features
cbMMunique   : number of reads with cell barcodes that map with mismatches to one barcode in the passlist
cbMMmultiple : number of reads with cell barcodes that map with mismatches to multiple barcodes in the passlist

exonicAS     : number of reads mapping antisense to annotated exons
intronicAS   : number of reads mapping antisense to annotated introns

These are only calculated for --soloFeatures GeneFull_Ex50pAS

exonic and intronic are only calculated for --soloFeatures GeneFull_Ex50pAS and/or GeneFull_ExonOverIntron

Multiple --soloFeatures options can be used at the same time, so if you want to use GeneFull for downstream analysis, but want to get separate intronic/exonic statistics, you can use --soloFeatures GeneFull GeneFull_Ex50pAS


question:

Thank you so much for your incredible work on this great tool! On the same topic could you please clarify the meaning of

countedU
countedM
nUMImulti
nGenesMulti


answer:

countedU : number of unique-gene reads that were used in counting UMIs (!= number of UMIs), i.e. reads with valid CB/UMI/gene
countedM : same for multi-gene reads
nUMImulti : number of UMI for multi-gene reads, if requested
nGenesMulti : number of genes supported by just multi-gene reads, if requested
